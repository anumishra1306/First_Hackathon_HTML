Media Tag

"Media tag" usually refers to the HTML elements used to include media: <audio>, <video>, <embed>, <iframe>. These are native HTML elements — no JavaScript or CSS required to include media files.

Integrating Audio Files

Use the <audio> element. Provide multiple <source> elements in different formats so more browsers can play the file.

Attributes

  • controls — show play/pause/seek UI provided by the browser.
  • autoplay — start playing automatically (often blocked by browsers; see autoplay section).
  • loop — restart after ending.
  • muted — start muted (useful when autoplaying, since many browsers require muted autoplay).

Example

<audio controls>
  <source src="audio/song.mp3" type="audio/mpeg">
  Your browser does not support the audio element. You can <a href="audio/song.mp3">download the audio</a>.
</audio>

Adding Controls to Audio

Add the controls attribute to let users play/pause/seek. If you omit controls, the audio will be invisible unless you add custom controls (requires JavaScript).

Integrating Video Files

The <video> element embeds video. Like audio, supply multiple <source>s and a fallback message.

Common attributes

  • controls — show native UI
  • width and height — sizing hints
  • poster — image shown before playback starts

Example

<video width="640" height="360" controls poster="images/preview.jpg">
  <source src="video/lecture.mp4" type="video/mp4">
  <source src="video/lecture.webm" type="video/webm">
  Your browser does not support the video tag. <a href="video/lecture.mp4">Download the video</a>.
</video>

Audio Formats

Use multiple formats for compatibility:

  • MP3 (audio/mpeg) — widely supported, good for music and spoken audio.
  • OGG (audio/ogg) — open format supported in modern browsers (useful fallback).
  • AAC / M4A (audio/mp4) — good quality, used on many platforms, but licensing can be a consideration.

Provide at least MP3 + OGG for best reach.

Why Proper Formats are Needed

Different browsers and devices support different codecs and container formats. Providing multiple formats increases the chance the user's browser can play your media. Also consider file size and network bandwidth when choosing formats and bitrates.

Video: Proper Formats

  • MP4 (H.264 video + AAC audio) — the most widely supported combination across browsers and devices.
  • WebM (VP8/VP9 + Vorbis/Opus) — modern open format; smaller files for comparable quality in many cases.
  • Ogg/Theora — older, less common now but still supported by some browsers.

Supply MP4 and WebM to cover almost all modern browsers.

Embedding YouTube Videos

Use an <iframe> to embed YouTube. YouTube provides an embed URL (choose the Share > Embed option on YouTube to get the exact URL). 

<!-- Replace VIDEO_ID with the video's id (e.g. dQw4w9WgXcQ) -->
<iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

Notes: You can add query parameters to the src, e.g. ?rel=0&showinfo=0 (YouTube parameters change over time — this is an example). The allowfullscreen attribute permits the user to expand the iframe to fullscreen.

Allow Fullscreen in Videos

For <iframe> embeds (like YouTube), use allowfullscreen or allow="fullscreen". For the native <video> element, browsers provide a fullscreen control when controls is present. 

<!-- YouTube iframe with fullscreen allowed -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID" frameborder="0" allowfullscreen title="Video"></iframe>

Multi-Page Websites

A multi-page website is simply multiple HTML files linked together. Example: index.html, about.html, contact.html. Use <a> to link pages.

Example 

/project-folder/
  index.html
  about.html
  contact.html
  /video/
    lecture.mp4
  /audio/
    song.mp3

Linking pages

<!-- in index.html -->
<nav>
  <a href="index.html">Home</a> 
  <a href="about.html">About</a> 
  <a href="contact.html">Contact</a>
</nav>

Footer Section

The footer usually contains copyright, contact info, and small navigation. In plain HTML: 

<footer>
  <p>&copy; 2025 Your Site Name. All rights reserved.</p>
</footer>

Header Section & Main Section

Use semantic elements <header> and <main> for structure and better accessibility. 

<header>
  <h1>Site Title</h1>
  <nav> <!-- simple nav links --> </nav>
</header>

<main>
  <h2>Article Title</h2>
  <p>Main content goes here.</p>
</main>

Autoplay Feature

<audio autoplay muted>
  <source src="audio/short.mp3" type="audio/mpeg">
</audio>

<video autoplay muted loop playsinline>
  <source src="video/clip.mp4" type="video/mp4">
</video>

Important: Modern browsers often block autoplay with sound. A common workaround is muted with autoplay (browsers generally allow muted autoplay). Always consider accessibility — autoplay can be disruptive to users.
