HTML Forms

HTML forms are used to collect user input. They consist of form elements like text fields, checkboxes, radio buttons, dropdowns, and buttons.

1. Basic Form Structure

The <form> tag defines a form. Its common attributes are:

  • action: URL where the form data is sent.
  • method: HTTP method to submit the form. Common values are get and post.
  • name: Name of the form.
  • id: Unique identifier.

Example:

<form  method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username"><br><br>

    <label for="email">Email:</label>
    <input type="email" id="email" name="email"><br><br>

    <input type="submit" value="Submit">
</form>

2. Common Form Input Types

  • text: Single-line text input.
  • password: Text input where characters are hidden.
  • email: Input for email addresses.
  • number: Input for numbers.
  • radio: Select one option from a group.
  • checkbox: Select multiple options.
  • file: Upload files.
  • submit: Button to submit the form.
  • reset: Button to reset the form fields.
  • button: Generic button for JS actions.
  • date, time, color, range: Special input types for modern browsers.

3. Form Labels and Attributes

The <label> tag defines a label for an input. Use the for attribute to link it to an input's id.

Example:

<label for="age">Enter your age:</label>
<input type="number" id="age" name="age" min="1" max="100">

4. Form Textareas and Dropdowns

<textarea> is used for multi-line text, and <select> creates dropdown lists.

Example:

<label for="bio">Your Bio:</label><br>
<textarea id="bio" name="bio" rows="4" cols="50"></textarea><br><br>

<label for="gender">Gender:</label>
<select id="gender" name="gender">
    <option value="male">Male</option>
    <option value="female">Female</option>
    <option value="other">Other</option>
</select>

5.HTML <form> Attributes

  1. action

    Defines where the form data will be sent after submission.

    <form action="submit.php"></form>
  2. method

    Defines how form data is sent to the server.

    • GET → Data visible in URL
    • POST → Data sent securely in body
    <form method="post"></form>
  3. target

    Defines where the result is shown.

    • _self → Same page (default)
    • _blank → New tab
    <form target="_blank"></form>
  4. name

    Gives a name to the form (used in JavaScript).

    <form name="regForm"></form>

    5. 6.HTML <input> Attributes

    1. type

      Defines the input type.

      <input type="email" name="userEmail">
    2. name

      Used to identify the field (sent to server).

      <input type="text" name="username">
    3. id

      Unique identifier for label/JS.

      <input type="text" id="user">
<label for="user">Enter Name</label>
    4. value

      Default value or button text.

      <input type="text" value="Default Name">
    5. placeholder

      Hint text.

      <input type="text" placeholder="Enter your name">
    6. required

      Mandatory field.

      <input type="email" required>
    7. disabled

      Cannot edit & not submitted.

      <input type="text" value="Disabled" disabled>
    8. maxlength / minlength

      Character limits.

      <input type="text" maxlength="10" minlength="3">
    9. size

      Visible width (characters).

      <input type="text" size="30">
